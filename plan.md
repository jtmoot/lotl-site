# Plan — Issue #2: Project foundation & deploy-ready pipeline

Branch: `feature/2-foundation` (off `docs/prd-ladies-on-the-links`, so PRD + design-reference + embed are present).
Method: scaffold first (setup, not TDD-able), then red-green TDD the two behavioral seams.

## Goal (from #2 acceptance criteria)
A deployable Astro static site whose **shell + both test harnesses** are in place, so slices #3–#11 parallelize behind it. No Cloudflare/DNS work here (that's #14, needs Stacey's 2FA).

---

## Phase 1 — Scaffold (setup, no tests)

1. `npm create astro@latest` into the repo root — **minimal/empty** template, TypeScript strict, no sample content.
2. Add integrations/deps:
   - `@astrojs/sitemap` (auto-sitemap for indexed pages)
   - `@astrojs/check` + `typescript`
   - dev: `@playwright/test` (rendered-page smoke seam)
   - `astro:content` (built in) for the schema seam — a tiny throwaway collection is defined here so the harness has something to assert; real collections land in #7/#12.
3. `.gitignore`: `node_modules/`, `dist/`, `.astro/`, `*.mov`, plus keep raw originals out (they live in `../`).
4. `astro.config.mjs`: `site: 'https://ladiesonthelinksgolf.com'`, sitemap integration, static output.
5. Fonts + tokens:
   - Google Fonts `Newsreader` (serif headlines) + `Geist` (sans body) via `<link>` in the base head.
   - CSS custom properties in a global stylesheet: `--forest #1A4A33`, `--cream #F6EFDF`, `--rose #DE809F`, `--blush #EBA9BE`, `--ink #1C1C22`; radii (pill 9999px, cards 2rem), section rhythm. (Garden-Industrial aesthetic per design-reference/DESIGN.md — inspiration, not literal.)

## Phase 2 — Brand assets (copy optimized derivatives only)
Copy from parent `../` into `src/assets/` (per asset-handling decision — optimized copies in repo, originals stay in `../`):
- `../logo-images/HiLogo.jpg` → wordmark (nav + footer)
- `../logo-images/golf-ball-logo.png` → `golf-ball-logo.png` (rename, no spaces) → OG/banner image

Hero image is **not** copied here — it belongs to the Home slice (#3); #2 ships a placeholder Home.

## Phase 3 — Base shell
- `src/layouts/Layout.astro`: head (title/description, OG image = golf-ball-logo, favicon, fonts, canonical), optional `noindex` prop (default indexable), slot for page content.
- `src/components/Nav.astro`: glassy/liquid-glass floating nav with wordmark + 9 items in order: **Home · Schedule/Book · Register · About/The Pro · Gallery · Stories · Merch · FAQ · Contact**. Tee sheet deliberately absent. Mobile hamburger.
- `src/components/Footer.astro`: tagline "For the love of golf.", **Facebook link** `https://www.facebook.com/Ladiesonthelinksgolfleague/`, North Hill address "29 Merry Avenue, Duxbury, MA 02332", **Legal → Privacy** link (`/privacy`, page itself built in #11).
- `src/pages/index.astro`: placeholder Home rendered through the shell (real content in #3).

## Phase 4 — TDD the two seams (red → green)

### Seam A — content-schema (highest priority)
Test (`src/content/` + a schema test): a valid entry parses; an entry missing a required field or with a bad reference **fails the build** with a clear error.
- RED: write the schema test asserting valid-parses / invalid-rejected against a sample collection.
- GREEN: define the sample collection schema (zod) so valid passes and malformed throws at build.
- This establishes the convention #7 (gallery) and #12 (stories) reuse.

### Seam B — rendered-page smoke (Playwright against built site)
Tests run against `astro build` output (`npm run build && playwright test`):
1. Home renders through the shell (status 200, core landmark present).
2. Nav contains exactly the 9 expected items **and excludes** any tee-sheet link.
3. Footer contains the Facebook link, the address, and the Privacy/Legal link.
4. Head emits the OG image + favicon; a `sitemap-index.xml` is generated by the build.
5. Default page does **not** emit `noindex`; the `noindex` Layout prop, when set, emits the directive (proves the mechanism #11 relies on).
- RED first (selectors fail), then build Phase 3 markup to GREEN.

## Phase 5 — Wire `npm test`
`package.json` scripts: `dev`, `build`, `preview`, `test` (runs schema test + `astro build` + playwright), `test:e2e`, `astro check`. `npm test` must run **both** seams.
